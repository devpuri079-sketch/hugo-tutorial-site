# Hugo によるサイト作成手順

---

## この手順の目的

この手順では、Hugo を使ってローカル環境にブログサイトを構築し、  
テーマの導入、記事の作成、画像の表示方法までを一通り体験できるようにする。

最終的に以下ができるようになる。

- Hugo プロジェクトの作成  
- テーマ（ananke）の導入  
- ローカルサーバーでのプレビュー  
- 記事の追加とドラフト管理  
- 画像の表示方法（static / PageBundle）

---

## 作成手順

### 1. Hugo プロジェクトを作成する

ターミナルを開き、任意のフォルダで以下を実行する。  
プロジェクトフォルダと設定ファイルが作成される。

```bash
hugo new site my-hugo-site
```

---

### 2. 作成されたフォルダをエディタで開く

VS Code などでプロジェクトフォルダを開く。  
`archetypes` 以外のフォルダは空なので、既存プロジェクトをコピーしたり、  
手動でフォルダを作成しても問題ない。

![プロジェクト画面](image1.png)

---

### 3. Git 管理を開始する

プロジェクトフォルダに移動し、以下を実行する。

```bash
git init
```

![git init](image2.png)  

Git 管理にすることで、次の手順で行うテーマのダウンロードが可能になる。  
また、将来的に GitHub へ push して公開するためにも必要となる。

---

### 4. テーマをダウンロードする（submodule）

公式チュートリアルでも使用されている ananke テーマを追加する。

```bash
git submodule add https://github.com/theNewDynamic/gohugo-theme-ananke.git themes/ananke
```

---

### 5. `hugo.toml` にテーマを設定する

以下を追記する。

```toml
theme = "ananke"
```

![テーマ設定](image3.png)

---

### 6. ローカルサーバーを起動する

```bash
hugo server
```

---

### 7. ブラウザで確認する

ターミナルに表示される URL（通常 http://localhost:1313/）を開く。  
Top ページが表示されていれば成功となる。

![Topページ](image4.png)

---

### 8. 記事を追加する

```bash
hugo new content/posts/my-first-post.md
```

以下のようにヘッダー部分だけのファイルが作成される。

![ファイル内容](image5.png) 

記事は `content` 配下に配置する。  
一般的には以下のような構成になる。

- ブログ記事 → `content/posts`  
- ドキュメント → `content/docs`

---

### 9. Markdown で本文を追記する

![ファイル内容編集](image6.png)

---

### 10. サーバーを -D（ドラフトモード）で起動する

```bash
hugo server -D
```

記事が表示されていることを確認する。

![ファイル内容編集](image7.png)

> **ドラフトモードについて**  
> - `draft = true`（下書き状態）の記事は通常の `hugo server` では表示されない  
> - `hugo server -D` を使うと draft の記事も表示される  
> - 公開したい記事は必ず `draft = false` に変更する（削除でも可）  
> - draft のまま公開し忘れるケースが多いため注意が必要となる

---

## 画像を表示する方法

画像の配置方法は 2 種類ある。

- static フォルダ方式  
- PageBundle（ページバンドル）方式

---

### 1. static フォルダから参照する方法

`static/images` フォルダを作成し、画像を配置する。  
Markdown では以下のように記述する。

```
![代替テキスト](/images/画像ファイル名)
```

![staticフォルダ](image8.png)

---

### 2. PageBundle（ページバンドル）方式

記事と画像を同じフォルダにまとめ、記事ファイル名を `index.md` にする。  
Markdown では以下のように記述する。

```
![代替テキスト](画像ファイル名)
```

![ページバンドル](image9.png)

---

## 画像の表示状況（テーマ ananke の場合）

**Top ページ**

![Top](image11.png)

**記事ページ**

![Top](image10.png)

ananke はブログ向けテーマのため、Top ページにサムネイルを表示するレイアウトになっている。  
その場合、static フォルダ方式でないと画像が表示されない。

ドキュメントページ用のテーマであれば Top ページにサムネイルを出すこともないため、  
記事と画像を一まとめにできる PageBundle のほうが適している場合もある。  
テーマによって仕様が異なるため、テーマの説明やサンプルサイトを確認することを推奨する。
